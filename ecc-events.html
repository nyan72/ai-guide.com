イベント駆動の解説 (Hooks & Plugins)

特定の動作をトリガーにして、検証やフォーマットを「自動発動」させる仕組み。

1. Hooks：ライフサイクル・イベント

Hooksは、ツールの実行前後やユーザーの入力時に割り込んで動作します。簿記の「仕訳のタイミング」のように、発生順序を理解することが重要です。

PreToolUse ツール実行直前（バリデーション、tmuxのリマインドなど）
↓
🛠️ Tool Execution (Bash, Edit, etc.)
↓
PostToolUse 実行完了直後（自動フォーマット、テスト実行、Lint確認）
↓
Stop Claudeの回答終了（console.logの消し忘れチェック等）

2. 実践設定例 (JSON)

tmuxのリマインド設定

長時間かかるコマンド（npm install 等）を実行しようとした際、tmux環境下でなければ警告を出します。

{
  // ツールがBash、かつコマンドが特定のパッケージマネージャーの場合
  "PreToolUse": [
    {
      "matcher": "tool == \"Bash\" && tool_input.command matches \"(npm|pnpm|yarn|pytest)\"",
      "hooks": [
        {
          "type": "command",
          "command": "if [ -z \"$TMUX\" ]; then echo '[Hook] セッション保護のためにtmuxの使用を検討してください' >&2; fi"
        }
      ]
    }
  ]
}

自動フォーマット (PostToolUse)

TypeScriptファイルを編集した直後に、自動的にPrettierを走らせます。

"PostToolUse": [
  {
    "matcher": "Edit && .ts/.tsx",
    "hooks": ["prettier --write"]
  }
]
