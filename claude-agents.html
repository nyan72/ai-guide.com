Best Practice for Agents & SubAgents

エージェントのオーケストレーション（運用・実行層）における重要指針

1. コンテキストの非対称性問題

サブエージェントは「要約」を返しますが、依頼側（Orchestrator）が持つ「真の目的」を理解していないため、重要な詳細が抜け落ちる傾向があります。

親エージェント (Orchestrator) サブエージェント (Sub-Agent)
【全体文脈を保持】
なぜこの調査が必要か、最終的に何を作りたいかの「背景」を知っている。 		【断片的知識のみ】
依頼された「クエリ」のみを処理する。背景を知らないため、要約の取捨選択を誤る。

2. 反復リトリーバル・パターン

この問題を解決するために、一度の回答で満足せず、以下のサイクルを回します（最大3回）。

1. 依頼：[クエリ] ＋ [全体目標(Objective)] をセットで投げる
2. 回答：サブエージェントが要約を返す
3. 評価：親が「十分か？」を判定
4. 追及：不足があれば「追加質問」を投げてソースを再確認させる

3. 逐次フェーズ (Sequential Phases)

複雑なタスクは以下の「フェーズ」に分割し、各工程の出力を次の入力に繋げます。

  • Phase 1: RESEARCH - パターンの特定と research-summary.md の作成
  • Phase 2: PLAN - 調査結果を読み、plan.md（実装計画）を作成
  • Phase 3: IMPLEMENT - 計画に基づきテスト駆動で実装
  • Phase 4: REVIEW - 全変更内容のコードレビュー
⚠️ 鉄則：フェーズを飛ばさない

各フェーズには独自の価値があります。中間出力は必ず「ファイル」として保存し、エージェント間で /clear を活用してコンテキストを新鮮に保ってください。

4. エージェント習得ティアリスト

難易度と効果に基づいた優先順位です。

Tier 1：即戦力・基本 (Direct Buffs)

  • Subagents: コンテキスト腐敗の防止。マルチエージェントより低複雑。
  • Metaprompting: プロンプトの安定性と前提条件の確認。
  • User Check: 実行前にユーザーへ詳細をヒアリングする。

Tier 2：上級・高難易度 (High Skill Floor)

  • Long-running agents: 長時間タスクの調整とトレードオフの理解が必要。
  • Parallel multi-agent: タスクの切り分けとマージが非常に困難。
  • Computer use: 未成熟な技術であり、制御の難易度が高い。
